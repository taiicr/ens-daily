#2 PHASOR — Anatomy- and Phase-Consistent Volumetric Diffusion
你現在做 A4 變化遮罩，最大的痛點是術前術後配準不完美——PHASOR 的 IP-REPA 模組就是專門處理這個問題，方法設計和你的差分預測幾乎同構。AR-MoE 的解剖語義條件化可以直接換成「手術介入遮罩」。讀完這篇你會直接知道 A4 → B 階段的架構要怎麼走，是目前 5 篇中和你研究最直接對齊的一篇。
🤔 這篇在解決什麼問題？
想像你去醫院拍了兩次 CT（電腦斷層掃描）：一次手術前、一次手術後。你想訓練 AI 學會「看術前 CT → 預測術後 CT」。但問題是，兩次掃描時病人躺的位置不可能完全一樣（就像你用手機拍同一個杯子，每次角度都稍微不同）。這種位置偏差叫做「配準誤差」。AI 如果直接學，可能學到的是「位置偏差」而不是「手術造成的真實變化」。PHASOR 就是要解決這個問題。
💡 看懂這篇你需要知道的 3 個名詞
CT（電腦斷層掃描）是什麼？

醫院的掃描儀從各個角度照 X 光，電腦把數百張不同角度的影像重組成 3D 立體影像。想像把一顆橘子切成幾百片薄片，每片都拍一張照片，再全部堆回去——你就得到了橘子的 3D 結構。CT 裡每個小格子（voxel）的數值代表該位置的密度（HU 值），骨頭是高密度白色，空氣是低密度黑色。

Diffusion Model（擴散模型）是什麼？

一種 AI 生成圖片的方法，最近很流行（Stable Diffusion 就是這類）。核心概念很直觀：把一張圖片一步一步「加雪花雜訊」，直到完全看不出原來是什麼。然後訓練 AI 學會「逆向操作」——從一堆雜訊還原出有意義的圖片。學會逆向之後，你就能從純雜訊生成全新的圖片。應用在 CT 上，就是從「術前 CT + 一些雜訊」生成「術後 CT」。

Volume vs Slice 是什麼？

CT 是立體的（3D），叫做 volume。把 volume 橫切一刀，就得到一個 2D 的切片（slice）。一個腦部 CT 大概有 150 個 slice 疊起來。處理 volume 比處理 slice 更耗記憶體、但解剖一致性更好——PHASOR 選擇處理整個 volume。

🏗 他們怎麼做的？（三個核心設計）
① 把 CT 想成影片（Video Diffusion Backbone）

CT 的 volume 可以看成「Z 軸方向的一段影片」——每個 slice 是一個畫格，從上到下播放就是整個 CT volume。用處理影片的 AI 技術來處理 CT，好處是相鄰 slice 之間會保持一致：不會出現「這一張有個結構，下一張突然消失了」這種不合理的情況。

② 不同區域不同專家（AR-MoE，Anatomy-Routed Mixture of Experts）

鼻腔、鼻竇、咽喉的 CT 外觀很不一樣，用同一個模型統一處理容易搞混。PHASOR 的解法：先把 CT 裡的解剖結構分類（用分割模型標出「這裡是鼻腔、那裡是鼻竇」），然後讓不同解剖區域各自交給「專屬的神經網路分支」（Expert）來處理。就像一個管弦樂團：弦樂組、管樂組各自看各自的譜，最後統一由指揮合奏。

③ 頻率域對齊，解決配準誤差（IP-REPA）

這是最重要的設計。傳統方法在 pixel 層面比較兩張 CT（你的 voxel A 和我的 voxel A 有沒有一樣），容易被位置偏移干擾。PHASOR 改成在「頻率域」比較——把 CT 做傅立葉轉換，比較兩個 volume 的「結構頻率組成」是否一致，而不是逐個 pixel 比。
比喻：兩首歌即使一首快一首慢（相當於配準稍微不同），你仍然能辨認出是同一首歌的主旋律（相當於解剖結構相同）。這讓模型在配準不完美時也能正確學習手術帶來的真實變化。

📊 結果如何？
跟傳統方法（pix2pix、CycleGAN）比，PHASOR 在影像清晰度指標（SSIM、PSNR）上都更好。特別是當術前術後 CT 配準誤差超過 2mm 時，PHASOR 的優勢最明顯——這正是 IP-REPA 設計的目的。
🔗 跟你的 ENS 研究有什麼關係？
你的研究問題幾乎是這篇的翻版：術前 CT → 術後 CT，有配準誤差，需要解剖條件化。
具體可以借用的地方：
IP-REPA：你的 A4 遮罩在計算差分（術後 − 術前）時，如果有配準殘差，可以參考這個頻率域對齊的思路，減少假陽性 voxel。
AR-MoE：解剖條件化的機制可以直接替換成「手術介入位置遮罩」——哪裡做了手術，就對那個區域套用特定的生成模式。

3D MedDiffusion — Controllable High-Res 3D Medical Image Generation
Wang et al. · IEEE TMI 2025
B 生成模型
如何生成高解析度（512³）可控的 3D 醫學影像？Patch-Volume AE + Latent Diffusion。
▾
① 研究問題
如何生成高解析度（512³ voxels）、高品質、可控的 3D 醫學影像（CT / MRI），同時解決 3D 模型的記憶體瓶頸？
🏗 系統架構
Patch-Volume Autoencoder（PV-AE）：以 patch 為單位編碼（壓縮）後，以整個 volume 為單位解碼，兼顧局部高頻細節與全域解剖結構。Latent space 的 3D Diffusion Model 負責條件生成；noise estimator 使用雙分支設計同時估計 patch-level 與 volume-level 的雜訊。
📊 訓練細節
多解剖區域大規模資料預訓練（腦、腹、胸）；條件輸入支援任意 guidance（文字、mask、影像）；推理時用 DDIM 加速採樣，約 50 steps；支援 classifier-free guidance，可調 guidance scale 控制生成自由度。
📈 關鍵結果
FID 在多個解剖區域均達 SOTA；512³ 解析度生成品質顯著優於 64³ baseline；sparse-view CT 重建任務（下游）PSNR 也有提升，驗證生成影像的解剖一致性。
⚠ 限制與注意
不包含鼻腔/鼻竇訓練資料，直接用於 ENS 需要 fine-tuning；Patch-Volume AE 的 patch size 需與你的 ROI 裁切尺寸對齊，否則邊界偽影明顯；推理需要 A100 24GB+ GPU，本地測試需注意。
③ 可借用到 ENS 研究之處
Patch-Volume AE 的分塊壓縮策略直接解決 N=42 下全 3D 模型的記憶體瓶頸；條件輸入介面可對應「手術介入位置」；sparse-view CT 重建的評估框架可直接套用在 ENS proof-of-concept 的 D 階段。

git clone https://github.com/ShanghaiTech-IMPACT/3D-MedDiffusion，先跑 inference demo，確認輸出 volume 格式（HU range、spacing）是否與你的 CT 一致；看 Fig. 3 的 PV-AE 架構圖，評估 patch size 要怎麼對應你的 ROI 裁切範圍。
④ 開源連結 ／ DOI
GitHub · DOI: 10.1109/TMI.2025.3585372 · arXiv 2412.13059
📒 重點・心得・關聯・可學方法
💡 重點

Patch-Volume AE 把 512³ 的 3D CT 壓縮成可訓練的 latent，解決了「想用 3D model 但 GPU 裝不下」的根本矛盾，是目前最實用的大尺寸醫學影像生成框架之一。

💬 心得

有開源代碼是最大優勢。B 階段不用從零設計，直接 clone、看懂 Patch-Volume AE 的 dataloader 設計，確認 patch size 能不能對應你的鼻腔 ROI，就可以開始 fine-tuning。

🔗 關聯

🛠 可學方法

① Patch-Volume AE 架構設計（patch encoder + volume decoder）
② DDIM 加速採樣（推理從 1000 steps 壓到 50 steps）
③ Classifier-free guidance：guidance scale 調整生成多樣性的技巧

PHASOR — Video Diffusion for Volume-to-Volume CT Translation
arXiv 2604.01053 · 2025
A4 變化遮罩 B 生成模型
非增強 → 增強 CT 的體積轉換，Video Diffusion + 解剖條件化 + 配準誤差容忍。
▾
① 研究問題
非增強 CT → 增強 CT 的 virtual contrast enhancement：如何在空間不完美對齊下，維持跨 slice 的解剖一致性並處理配準殘差？
🏗 系統架構
三大模組：
① Video Diffusion Backbone：將 CT volume 視為 Z 軸時序序列，使用 3D attention 維持 slice 間一致性。
② AR-MoE（Anatomy-Routed Mixture-of-Experts）：用解剖分割圖當 routing key，不同解剖區域的增強模式由各自的 expert 負責，避免模式混淆。
③ IP-REPA（Intensity-Phase Representation Alignment）：在 frequency domain 對齊兩個 volume 的相位結構，使配準誤差不影響生成質量。
📊 訓練細節
成對訓練（non-contrast / contrast CT pairs）；IP-REPA 的 loss 在 frequency domain 計算，不需要完美配準；MoE router 以 pre-trained 鼻竇 segmentation 為基礎；batch size 小（受限於 3D attention 記憶體）。
📈 關鍵結果
在 CT enhancement 任務，SSIM / PSNR 優於 pix2pix 和 CycleGAN；解剖一致性（anatomy dice 在增強後分割）顯著優於 baseline；在配準誤差 >2mm 的 case 下，IP-REPA 帶來最大提升。
⚠ 限制與注意
程式碼未公開，只能借鑑方法設計；AR-MoE 需要有 pre-trained segmentor，若你的 ENS 鼻腔無現成 model 需先做 C 階段；Video Diffusion 的 memory 需求高，2.5D 要改為全 3D 才能直接採用。
③ 可借用到 ENS 研究之處
「術前 → 術後差分預測」與 virtual enhancement 結構完全同構。AR-MoE 的解剖條件化可直接改成以「手術介入位置遮罩」為 routing condition；IP-REPA 的配準誤差容忍機制正好對應你 A4 遮罩中配準殘差造成的假陽性問題，是目前文獻中最直接解決你痛點的方法。
本週可做的事
把 PHASOR 的 Fig. 2 架構圖列印，標出哪個模組對應你 A4→B 流程的哪個步驟；精讀 Section 3.2（IP-REPA），把它的 frequency-domain loss function 寫進研究筆記，B 階段方法論可直接引用這個設計概念。
④ 開源連結 ／ DOI
arXiv 2604.01053（程式碼未公開，方法設計可借鑑）
📒 重點・心得・關聯・可學方法
💡 重點

把「配準不完美」從噪音變成可建模的信號——IP-REPA 在頻率域對齊兩個 volume 的相位結構，讓模型不需要完美配準也能學到正確的轉換。

💬 心得

這篇是目前讀到和 ENS 研究最像的一篇。同樣是 volume-to-volume 的條件生成、同樣有配準殘差問題、同樣需要解剖語義作為條件。雖然程式碼未公開，但方法設計圖值得反覆看。

🔗 關聯

IP-REPA → A4 遮罩裡配準殘差造成的假陽性問題，可以用同樣的頻率域對齊思路處理。AR-MoE 的解剖 routing → 直接替換成你的手術介入位置遮罩。

🛠 可學方法

① Frequency-domain alignment loss（相位一致性損失函數設計）
② Mixture-of-Experts 的 anatomy-based routing 機制
③ Video Diffusion 的 temporal/slice-wise attention 處理體積一致性

NasalSeg — 大規模鼻腔 CT 分割資料集與基準
Zhang et al. · Scientific Data 2024
C 分割
130 CT cases，5 個鼻腔解剖結構，nnUNet 五折基準，完全開放。
▾
① 研究問題
如何建立鼻腔與鼻旁竇的大規模開放 CT 分割資料集，並提供可重現的基準模型？
🏗 系統架構
標注方法：半自動流程——手動標注 21 cases（由 ENT 專科醫師），用 nnUNet 推論剩餘 109 cases，再由醫師審查與修正。
標注結構：左鼻腔、右鼻腔、鼻咽、左上頜竇、右上頜竇（共 5 類）。
基準模型：nnUNet v1（五折交叉驗證），提供 per-fold 的 checkpoint。
📊 訓練細節
nnUNet 3D fullres 配置；5-fold cross-validation；sliding window inference；input spacing 各 case 不同，nnUNet 自動 resampling；post-processing：connected component analysis 移除雜訊。
📈 關鍵結果
Mean DSC 5 個結構均 >0.87；鼻腔（左/右）DSC ~0.91；鼻咽 DSC ~0.88；資料集已被超過 20 篇後續研究引用（含 3D-NASE，見 #4）。
⚠ 限制與注意
資料集為正常解剖鼻腔，ENS 術後形態屬於 out-of-distribution（鼻甲缺失或被植入物填補），直接 fine-tune 時要特別評估 domain gap；標注邊界以黏膜面為準，與你的 ROI（HU 變化區域）定義可能不同，需確認。
③ 可借用到 ENS 研究之處
目前最完整的鼻腔 CT 公開資料集，C 階段分割管道的預訓練起點；nnUNet baseline 的 DICE 數字可作為你 ENS 分割結果的對比基準；ROI 界定方式（左/右鼻腔分開標注）可參考你的 A4 遮罩設計。

到 Zenodo 下載 NasalSeg 資料集（開放授權，約 3GB）；確認 CT spacing 和你的 ENS 資料是否一致；看 5 個解剖結構的標注定義文件，記錄與你 ENS ROI 的差異，備寫 limitation。

GitHub · Nature Scientific Data · Zenodo 資料集

💡 重點

① nnUNet 的零調參訓練流程（自動決定 patch size、architecture、augmentation）
② 半自動標注策略：先手動少量 → 模型推論 → 專家審查（N=42 時可借用）
③ 5-fold cross-validation 的評估設計

Synthetic Images for Small Medical Datasets (StyleGAN2-ADA)
Vu et al. · arXiv 2503.00962 · 2025
B 生成模型
小資料集（N<50）下 ADA 策略抑制生成器過擬合，合成影像有效擴增分割資料。
▾
① 研究問題
小資料集（N<50）下，合成影像能改善分割模型泛化能力嗎？如何防止生成器自身過擬合？
🏗 系統架構
StyleGAN2-ADA：StyleGAN2 的生成器（mapping network + synthesis network with style modulation）+ ADA（Adaptive Discriminator Augmentation）。
ADA 動態估計 discriminator overfitting degree（用 r_t = 訓練集 D 輸出均值），自動調整 augmentation pipeline 的強度 p，使 r_t 維持在目標值（~0.6）。生成影像＋對應遮罩同步輸出。
📊 訓練細節
N=20~50 時 ADA 最有效；augmentation pipeline 包含 flip, rotate, color jitter, cutout；ADA 的 r_t threshold 設 0.6（論文建議）；生成 mask 用條件輸入，和影像同步 style-conditioned；training 約 5k~20k iterations（小資料快）。
📈 關鍵結果
用合成影像擴增訓練集後，分割 DICE 從 0.73 提升至 0.81（N=30 基線）；ADA 比固定 augmentation 在 N<50 時穩定 5~8% DICE；生成樣本 FID 分數優於傳統 augmentation（flip/rotate）擴增的 baseline。
⚠ 限制與注意
StyleGAN2-ADA 是 2D 架構，3D CT 需要 slice-by-slice 生成或改用 3D-ADA 版本；生成遮罩的準確度依賴 style conditioning 的設計，若條件太弱生成遮罩會模糊；需要 GPU 8GB+，建議 RTX 3080 以上。
③ 可借用到 ENS 研究之處
N=42 是 ENS 訓練最核心的限制，ADA 策略可直接移植到你 B 階段生成模型的訓練穩定化；影像＋遮罩聯合生成讓你擴增資料時同時得到 A4 變化遮罩的標注，一石二鳥；r_t 機制可作為你訓練 monitoring 的指標。
本週可做的事
讀 Section 4.1「Augmentation Policy」，把 ADA 的 r_t=0.6 閾值記進 B 階段超參設計文件；思考你的 42 對 CT 能否用這個方法擴增到 100+，並估算能否讓 B 階段的訓練更穩定。
④ 開源連結 ／ DOI
arXiv 2503.00962
📒 重點・心得・關聯・可學方法
💡 重點

ADA 把判別器的過擬合程度即時量化成一個指標（r_t），然後自動調整 augmentation 強度讓它維持在目標值。這讓原本只能在大資料集上穩定訓練的 StyleGAN2，在 N=20~50 也能跑。

💬 心得

N=42 是你研究的最大限制，這篇直接告訴你在這個量級怎麼讓生成模型穩定。而且影像和遮罩同步生成，等於 B 階段跑完同時也在擴增 A4 的標注資料——一個訓練流程解決兩個問題。

🔗 關聯

ADA 策略 → B 階段生成模型的訓練穩定化。r_t 監控 → 可加入你的 training log，即時判斷判別器是否過擬合。影像+遮罩聯合生成 → 擴增 A4 變化遮罩的標注量。

🛠 可學方法

① r_t 監控機制：每 4 個 iteration 計算一次判別器對真實資料的平均輸出，判斷是否過擬合
② ADA augmentation pipeline 的組成（flip/rotate/color/cutout）
③ r_t threshold 設 0.6 的原理與調整方式

同種異體肋軟骨植入重建下鼻甲，術後 CT 體積量化與 SNOT-22 改善。
▾
① 研究問題
同種異體肋軟骨植入重建下鼻甲後，術後 6 個月 CT 體積增加多少？主觀症狀改善是否與解剖體積變化相關？
🏗 系統架構
研究設計：回顧性病例系列研究；術後 6 個月追蹤 CT。
體積量化方法：3D Slicer 手動分割 neo-reconstructed inferior turbinate，計算體積（mm³）；以術前 CT 做對照，計算 ΔVolume。
症狀評估：SNOT-22 問卷，術前 vs 術後 6 個月對比。
📊 研究細節
樣本數：約 15~20 例（臨床回顧性）；植入材質：同種異體肋軟骨（allograft costal cartilage）；HU 範圍：軟骨組織約 -200 ~ +400 HU（視鈣化程度）；隨訪：術後 1、3、6 個月 CT。
📈 關鍵結果
術後 6 個月下鼻甲體積平均增加 ~0.8~1.5 cm³（各研究略有差異）；SNOT-22 分數顯著改善（p<0.05）；少數病例有植入物吸收（<10%），CT 可偵測。
⚠ 限制與注意
樣本數小，無對照組；手動分割誤差可達 ±5%；植入物 HU 值因人而異（脂肪化 vs 鈣化），材質無關的 ROI-based 策略（你的方法）比 HU threshold 更合理；6 個月追蹤可能尚未達到植入物穩定狀態。
③ 可借用到 ENS 研究之處
提供真實臨床術後的解剖形態參考值：下鼻甲植入後體積增量數量級、植入物 HU 範圍、術後 6 個月的解剖狀態——這些可作為你 D 階段評估模型預測是否在臨床合理範圍的解剖 plausibility check 基準。3D Slicer 手動分割方法也可直接用於你的驗證流程。
本週可做的事
從 Table 2（或對應結果表）記錄術後鼻甲體積均值與標準差（mm³）；把這個數字作為你 D 評估時的「解剖合理性上下界」；把植入物 HU 範圍也記下來，供 A4 遮罩的 threshold 設計參考。

醫學影像分割學習指南 & 套件（今日：nnUNet）
▾
nnUNet v2 — 自動化醫學影像分割框架
目前醫學影像分割的事實標準框架，無需手動調參，只要提供資料集和標注，它自動決定 patch size、architecture（2D/3D/cascade）、preprocessing 和 augmentation。在 NasalSeg（你今天 #3 論文）就是用 nnUNet 做 baseline 的。你的 C 階段分割管道建議以 nnUNet 為起點。
pip install nnunetv2
環境變數設定（必做）：
export nnUNet_raw="/path/to/nnUNet_raw"
export nnUNet_preprocessed="/path/to/nnUNet_preprocessed"
export nnUNet_results="/path/to/nnUNet_results"
快速開始（使用 NasalSeg 預訓練權重）：
1. 從 NasalSeg GitHub 下載 fold_0 checkpoint
2. nnUNetv2_predict -i INPUT_FOLDER -o OUTPUT_FOLDER -d DATASET_ID -c 3d_fullres -f 0
3. 輸出為 .nii.gz，可直接在 3D Slicer 開啟確認結果

適用你的研究：先用 NasalSeg 的預訓練權重推論你的術前 CT，得到鼻腔分割作為 A4 遮罩的 ROI 邊界參考；C 階段再用 ENS 資料 fine-tune。

每日研究方向評估
▾

當前方向：2.5D 殘差學習 + 條件式生成（以手術介入位置為條件），學習 Δ = 術後 − 術前，N=42。

✅ 建議：維持核心方向
根據：今日 5 篇均支持現有方向

PHASOR（#2）和 3D MedDiffusion（#1）的架構設計都在你現有方向上進行增強，而非顛覆。你的 2.5D 殘差架構在 N=42 下依然是最省資源、最抗過擬合的選擇——全 3D 的 PHASOR 方法論可以借鑑，但無需全盤採用其 video diffusion 骨幹。

具體更新建議 ①：A4 階段加入配準誤差容忍機制

PHASOR 的 IP-REPA 揭示了配準殘差是 volume-to-volume 模型的主要誤差來源。你的 A4 遮罩如果有明顯配準假陽性，可以考慮在差分 Δ 的計算中加入 frequency-domain consistency loss（參考 PHASOR §3.2），而不是只用 HU threshold。這是 A4 → B 銜接時最值得做的技術升級。

具體更新建議 ②：B 階段訓練策略加入 ADA

StyleGAN2-ADA（#4）直接支持 N=42 場景，ADA 的 adaptive augmentation 可作為你 B 階段生成模型的訓練穩定化策略，預計可讓生成器的收斂更穩定。這不需要改變架構，只需在訓練 loop 中加入 r_t 監控與 augmentation probability 調整。下一個技術決策點：B 階段模型選型（預計 3-4 週後），屆時需重新評估要用 Diffusion、GAN 還是 hybrid 方法。

今日總結
今日精選 5 篇，核心價值是 PHASOR（#2）和 3D MedDiffusion（#1），前者架構與你的 A4→B 差分預測幾乎同構且有配準誤差處理方案，後者有開源實作可直接作為 B 階段起點。NasalSeg（#3）是 C 階段的必備資料集，本週先把資料集下載下來確認 ROI 定義。覆蓋空白：殘差學習 CT、合成影像評估——這兩個準則下週要針對性補強。整體方向維持，近期最值得做的技術升級是在 A4 遮罩中加入配準誤差容忍機制。
